Configuración de Accesos

Cómo obtener y configurar las credenciales para que email-mcp pueda enviar emails desde tu cuenta de Microsoft 365 / Outlook.

✅ Sin Azure AD requerido ⏱ ~5 minutos 📧 Microsoft 365 / Outlook
00 Diagnóstico previo

¿Tienes MFA activo en tu cuenta?

MFA (autenticación de dos factores) cambia cómo debes configurar la contraseña. Elige tu caso:

🔓

No tengo MFA / No sé

Entras a Outlook solo con email + contraseña, sin que te pida código del teléfono ni la app Authenticator.

→ Sigue la Ruta A (más sencilla)
🔐

Sí tengo MFA activo

Al entrar a Outlook te pide un código de la app Microsoft Authenticator, un SMS, o una notificación en el teléfono.

→ Sigue la Ruta B (contraseña de app)
💡

¿Cómo verifico si tengo MFA? Ve a account.microsoft.com → Seguridad → Opciones de seguridad avanzadas. Si ves "Verificación en dos pasos: Activada", estás en la Ruta B.

Ruta A
Sin MFA — contraseña directa
1
Anota tu email y contraseña de Outlook
Es la misma contraseña que usas para entrar a Outlook en el navegador o en la app de escritorio.
No necesitas hacer nada más en la web. Ya tienes todo lo necesario.
2
Crea el archivo .env
En la carpeta email-mcp/ crea un archivo llamado exactamente .env (con el punto al inicio):
# email-mcp/.env SMTP_USER=ldominguez@atisa.com SMTP_PASS=tu_contraseña_de_outlook FROM_EMAIL=ldominguez@atisa.com
⚠️

Nunca compartas este archivo. Está en .gitignore y no se sube al repositorio, pero no lo envíes por chat ni email.

3
Actualiza .mcp.json con la contraseña
Abre email-mcp/.mcp.json y rellena el campo SMTP_PASS:
{ "mcpServers": { "email-mcp": { "command": "node", "args": ["/home/llopez/.../email-mcp/src/index.js"], "env": { "SMTP_USER": "ldominguez@atisa.com", "SMTP_PASS": "tu_contraseña_aquí", ← rellena esto "FROM_EMAIL": "ldominguez@atisa.com" } } } }
4
Verifica que funciona
Desde la terminal, en la carpeta email-mcp/, ejecuta:
node src/index.js
Debes ver: [email-mcp] Servidor iniciado en modo SMTP.
Presiona Ctrl+C para detenerlo. Si ves ese mensaje, la configuración es correcta — Claude Code lo arrancará automáticamente.
✅

Ruta A completa. Continúa con el Manual de Claude Code para terminar la configuración.

Ruta B
Con MFA — contraseña de aplicación

Con MFA activo, tu contraseña normal no funciona para SMTP. Microsoft bloquea esto por seguridad. La solución es una contraseña de aplicación: una clave especial de 16 letras que Microsoft genera solo para apps que no soportan MFA.

ℹ️

¿Qué es una contraseña de aplicación? Es una contraseña separada que se usa solo para esta app. No reemplaza tu contraseña principal. Si la comprometes, puedes revocarla sin cambiar nada más.

1
Ve a la configuración de seguridad de tu cuenta Microsoft
Abre el navegador y ve a:
https://account.microsoft.com/security
Inicia sesión con tu cuenta ldominguez@atisa.com si te lo pide.
2
Abre las opciones de seguridad avanzadas
En la página de Seguridad busca y haz clic en:
Seguridad› Opciones de seguridad avanzadas
O busca directamente el enlace "Opciones de seguridad avanzadas" en la página.
3
Crea una nueva contraseña de aplicación
En la sección "Contraseñas de aplicación" haz clic en:
Contraseñas de aplicación› Crear una nueva contraseña de aplicación
Microsoft generará automáticamente una contraseña de 16 caracteres como esta:
abcd efgh ijkl mnop ← ejemplo, la tuya será diferente
⚠️

Cópiala en este momento. Microsoft solo la muestra una vez. Si la cierras sin copiarla, tendrás que generar otra.

4
Crea el archivo .env con esa contraseña
Usa la contraseña de aplicación sin espacios como valor de SMTP_PASS:
# email-mcp/.env SMTP_USER=ldominguez@atisa.com SMTP_PASS=abcdefghijklmnop ← sin espacios FROM_EMAIL=ldominguez@atisa.com
5
Actualiza .mcp.json
Igual que la Ruta A, rellena SMTP_PASS en email-mcp/.mcp.json con la contraseña de aplicación (sin espacios).
6
Verifica que funciona
Desde la terminal en la carpeta email-mcp/:
node src/index.js
Debes ver [email-mcp] Servidor iniciado en modo SMTP. — eso confirma que la contraseña es correcta y el servidor arranca bien.
✅

Ruta B completa. Continúa con el Manual de Claude Code para terminar la configuración.

⚠️ Caso especial

¿Y si IT deshabilitó SMTP AUTH en mi cuenta?

Algunos administradores de Microsoft 365 deshabilitan SMTP AUTH por política de seguridad. Si el servidor da error 535 o "AUTH command not accepted", este es tu caso.

1
Cómo saber si es este caso
Cuando el error al intentar enviar diga alguno de estos mensajes:
SMTP 535: 5.7.139 Authentication unsuccessful... SMTP 503: AUTH command not accepted 5.7.57 SMTP; Client was not authenticated
2
Qué pedirle a IT (es un cambio de 30 segundos para ellos)
Manda este mensaje a tu administrador de Microsoft 365:
Hola, necesito que habilites "SMTP AUTH" para mi cuenta ldominguez@atisa.com en el centro de administración de Microsoft 365. Ruta: Centro de administración M365 → Usuarios → mi cuenta → Correo → Configuración de correo → Envío de correo autenticado (SMTP) → Activar Es para una herramienta de automatización interna.
3
Alternativa: Graph API (si IT no puede ayudar pronto)
Si necesitas avanzar sin esperar a IT, puedes pedir a un administrador de Azure AD que cree un App Registration con permisos Mail.Send y Mail.ReadWrite. Luego configura TENANT_ID, CLIENT_ID y CLIENT_SECRET en el .env en lugar de SMTP_USER y SMTP_PASS. El servidor detectará automáticamente el modo Graph API y usará funcionalidad completa.
FAQ Errores comunes

¿Algo no funciona?

❌ Error: "Authentication unsuccessful" o código 535
Contraseña incorrecta. Verifica que SMTP_PASS sea exactamente la contraseña de Outlook o la contraseña de aplicación. Si tienes MFA activo y usaste tu contraseña principal, cambia a contraseña de aplicación (Ruta B). Asegúrate de no incluir espacios ni comillas alrededor del valor en el archivo .env.
❌ Error: "Timeout de conexión SMTP (30s)"
El servidor SMTP no es accesible. Puede ser: (1) estás en una red que bloquea el puerto 587 (algunas VPNs o redes corporativas hacen esto), (2) el host es incorrecto. Verifica que SMTP_HOST=smtp.office365.com y que tienes acceso a internet en la terminal (prueba ping smtp.office365.com).
❌ Error: "SMTP 503: AUTH command not accepted"
SMTP AUTH está deshabilitado por IT en tu cuenta. Sigue el Caso Especial descrito arriba para pedir a tu administrador que lo habilite para tu usuario.
❌ El servidor arranca pero el email no llega
Revisa la carpeta de Spam del destinatario. Los primeros envíos pueden ir a spam hasta que el receptor marque el remitente como seguro. También verifica en Elementos enviados de Outlook que el email efectivamente salió.
❌ El archivo .env no se carga
El archivo debe llamarse exactamente .env (con el punto) y estar en la raíz de email-mcp/, no dentro de src/. Verifica con ls -la email-mcp/ — debe aparecer en la lista. Si usas Windows y el explorador de archivos no te deja crear archivos que empiezan con punto, créalo desde la terminal con touch email-mcp/.env.
