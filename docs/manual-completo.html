email-mcp

01 Parte 1 · Credenciales

¿Tienes MFA (doble factor) activo?

MFA significa que para entrar a tu cuenta Microsoft necesitas además de la contraseña un código del teléfono, una notificación en la app Authenticator, o un SMS. Esto cambia cómo debes configurar la contraseña para SMTP.

🔓

Sin MFA — acceso solo con contraseña

Entras a Outlook solo con email + contraseña. El sistema no te pide ningún código extra.

→ Sigue la Ruta A (3 pasos)
🔐

Con MFA activo

Al entrar te pide un código de la app Authenticator, un SMS, o una notificación push en el teléfono.

→ Sigue la Ruta B (contraseña de app)
💡

¿Cómo verifico si tengo MFA? Entra a account.microsoft.com → Seguridad → Opciones de seguridad avanzadas. Si ves "Verificación en dos pasos: Activada", estás en la Ruta B.

Ruta A

Sin MFA — contraseña directa

1
Ten a mano tu contraseña de Outlook
Es la misma que usas para entrar al correo en el navegador o en la app de escritorio. No necesitas hacer nada en la web.
2
Crea el archivo .env
Dentro de la carpeta email-mcp/ crea un archivo llamado exactamente .env (con el punto al inicio). Escribe esto adentro:
email-mcp/.env
SMTP_USER=ldominguez@atisa.com SMTP_PASS=tu_contraseña_de_outlook FROM_EMAIL=ldominguez@atisa.com
⚠️

Nunca compartas este archivo. No lo subas a git, no lo envíes por chat. Contiene tu contraseña en texto plano.

3
Rellena SMTP_PASS en el archivo .mcp.json
Abre email-mcp/.mcp.json y pon tu contraseña en el campo indicado:
email-mcp/.mcp.json
{ "mcpServers": { "email-mcp": { "command": "node", "args": ["/home/llopez/tablet/Documents/home-Termux/email-mcp/src/index.js"], "env": { "SMTP_USER": "ldominguez@atisa.com", "SMTP_PASS": "tu_contraseña_aquí", ← esto "FROM_EMAIL": "ldominguez@atisa.com" } } } }
4
Prueba que arranca sin errores
En la terminal, desde la carpeta email-mcp/:
node src/index.js
Debes ver en la terminal:
[email-mcp] Planificador iniciado (chequeo cada 30s) [email-mcp] Servidor iniciado en modo SMTP. Esperando conexión de Claude Code...
Presiona Ctrl+C para detenerlo. Si aparece ese mensaje, la Parte 1 está completa.
✅

Ruta A completa. Pasa directamente a la Parte 2 — Claude Code.

Ruta B

Con MFA — contraseña de aplicación

ℹ️

Por qué tu contraseña normal no funciona con MFA: SMTP es un protocolo que no soporta el segundo factor. Microsoft bloquea la autenticación básica cuando MFA está activo. La solución oficial es una contraseña de aplicación: una clave de 16 letras generada especialmente para apps sin soporte de MFA.

1
Ve a la configuración de seguridad de Microsoft
Abre el navegador y ve a esta dirección:
https://account.microsoft.com/security
Inicia sesión con tu cuenta ldominguez@atisa.com.
2
Abre las opciones de seguridad avanzadas
Busca y haz clic en el enlace:
Seguridad › Opciones de seguridad avanzadas
3
Crea una nueva contraseña de aplicación
En la sección "Contraseñas de aplicación":
Contraseñas de aplicación › Crear una nueva contraseña de aplicación
Microsoft genera automáticamente algo como:
abcd efgh ijkl mnop ← ejemplo, la tuya será diferente
⚠️

Copia la contraseña en este momento. Microsoft solo la muestra una vez. Si cierras la pantalla sin copiarla, deberás generar una nueva.

4
Crea el archivo .env con esa contraseña
Usa la contraseña de aplicación sin espacios en SMTP_PASS:
email-mcp/.env
SMTP_USER=ldominguez@atisa.com SMTP_PASS=abcdefghijklmnop ← sin espacios FROM_EMAIL=ldominguez@atisa.com
5
Rellena también el .mcp.json
Igual que en la Ruta A, pon la contraseña de aplicación (sin espacios) en el campo SMTP_PASS de email-mcp/.mcp.json.
6
Verifica que arranca
node src/index.js
Debes ver [email-mcp] Servidor iniciado en modo SMTP. — eso confirma que la contraseña de aplicación es válida.
✅

Ruta B completa. Pasa a la Parte 2 — Claude Code.

⚠ Caso especial

¿IT deshabilitó SMTP AUTH en tu cuenta?

Algunos administradores de Microsoft 365 deshabilitan SMTP AUTH por política. Si obtienes el error 535 o AUTH command not accepted aunque tu contraseña sea correcta, este es tu caso.

1
Identifica el error
SMTP 535: 5.7.139 Authentication unsuccessful... SMTP 503: AUTH command not accepted 5.7.57 SMTP; Client was not authenticated
2
Pídele a IT que habilite SMTP AUTH para tu cuenta
Es un cambio de 30 segundos para el administrador. Copia y pega este mensaje:
--- Mensaje para IT --- Hola, necesito que habilites "SMTP AUTH" para mi cuenta ldominguez@atisa.com en el centro de administración de M365. Ruta: Centro de administración M365 → Usuarios → mi usuario → Correo → Configuración de correo → "Envío de correo autenticado (SMTP)" → Activar Es para una herramienta de automatización interna. Gracias.
3
Alternativa mientras esperas: Graph API
Si necesitas avanzar sin esperar a IT, pide a un admin de Azure AD que cree un App Registration con permisos Mail.Send + Mail.ReadWrite. Luego en el .env usa TENANT_ID, CLIENT_ID, CLIENT_SECRET en lugar de SMTP_USER y SMTP_PASS. El servidor detecta el modo automáticamente.
02 Parte 2 · Claude Code

Cómo funciona el archivo .mcp.json

Claude Code busca archivos .mcp.json al arrancar. Si encuentra uno, lanza los servidores MCP listados ahí como procesos en segundo plano y conecta las herramientas al asistente.

📁 Nivel de proyecto

Archivo: email-mcp/.mcp.json

Activo cuando abres Claude Code desde email-mcp/ o desde su carpeta padre home-Termux/.

✓ Ya existe y tiene la estructura correcta

🏠 Nivel de workspace

Archivo: home-Termux/.mcp.json

Activo desde cualquier subcarpeta: FinancalFreeHome/, naves-industriales-ai/, etc. Necesita la contraseña.

⚠ Rellenar SMTP_PASS
💡

Configura ambos. Así puedes enviar emails desde cualquier proyecto del workspace, no solo desde la carpeta de email-mcp.

Registrar el servidor — paso a paso

1
Confirma que email-mcp/.mcp.json tiene la contraseña
Ábrelo y verifica que SMTP_PASS no está vacío. Debe verse así:
"SMTP_USER": "ldominguez@atisa.com", "SMTP_PASS": "tu_contraseña_real", ← no debe estar vacío "FROM_EMAIL": "ldominguez@atisa.com"
⚠️

La ruta en args debe ser absoluta: /home/llopez/tablet/Documents/home-Termux/email-mcp/src/index.js. Las rutas relativas no funcionan en MCP.

2
Rellena también el .mcp.json del workspace raíz
Abre /home/llopez/tablet/Documents/home-Termux/.mcp.json y pon la misma contraseña en SMTP_PASS.
3
Abre Claude Code desde el workspace
cd /home/llopez/tablet/Documents/home-Termux claude
Claude Code leerá el .mcp.json automáticamente y arrancará email-mcp en segundo plano. No necesitas hacer nada más para iniciarlo.

04 Referencia

Solución de problemas

❌ /mcp no muestra email-mcp
Causa 1: Abriste Claude Code desde una carpeta que no tiene .mcp.json en su árbol. Solución: abre siempre desde home-Termux/.

Causa 2: Error de sintaxis en el .mcp.json. Verifica con:
node -e "JSON.parse(require('fs').readFileSync('email-mcp/.mcp.json','utf8'))"
Si no muestra error, el JSON está bien.

Causa 3: La ruta en args no existe. Verifica:
ls /home/llopez/tablet/Documents/home-Termux/email-mcp/src/index.js
❌ email-mcp aparece en rojo / "failed to start"
Corre el servidor manualmente para ver el error exacto:
node /home/llopez/tablet/Documents/home-Termux/email-mcp/src/index.js

Los errores más frecuentes: SMTP_PASS vacío en .mcp.json, ruta incorrecta, o permisos de archivo. El error aparecerá en pantalla con el mensaje exacto.
❌ Error "Authentication unsuccessful" (código 535) al enviar
Contraseña incorrecta. Verifica que SMTP_PASS no tenga espacios extras ni comillas. Si tienes MFA activo y usaste tu contraseña principal, necesitas una contraseña de aplicación (Ruta B). Revisa que el valor en .mcp.json y en .env sea idéntico.
❌ Timeout de conexión SMTP (30 segundos)
El servidor smtp.office365.com:587 no es alcanzable. Posibles causas: (1) estás en una red VPN o corporativa que bloquea el puerto 587, (2) no hay conexión a internet desde la terminal. Prueba: curl -v telnet://smtp.office365.com:587 — si no conecta, el puerto está bloqueado.
❌ El email programado no se envió a la hora indicada
El scheduler solo funciona mientras Claude Code está abierto y conectado al MCP. Si cerraste Claude Code, el proceso se detuvo. Los jobs quedan guardados en queue.json y se enviarán la próxima vez que Claude Code esté activo y llegue su horario.
❌ El email llega con caracteres raros en el asunto (tildes, ñ)
El asunto está codificado en UTF-8 con RFC 2047. Si tu cliente de correo no lo decodifica correctamente, prueba actualizar el cliente. En Outlook Desktop y Outlook Web esto funciona correctamente. En clientes más antiguos puede aparecer como =?UTF-8?B?...?=.
❌ "Los borradores requieren Microsoft Graph API" al usar email_borrador
Esto es esperado en modo SMTP. SMTP es un protocolo de solo envío — no puede crear borradores en el buzón. Para usar email_borrador necesitas configurar el modo Graph API con TENANT_ID, CLIENT_ID y CLIENT_SECRET. Ver la sección "SMTP bloqueado por IT" para la alternativa.
