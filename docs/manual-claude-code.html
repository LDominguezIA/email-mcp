Lista de verificación

Asegúrate de tener estos puntos antes de continuar:

01 Cómo funciona el registro MCP

¿Qué hace el archivo .mcp.json?

Claude Code busca archivos .mcp.json en dos lugares. Según dónde lo pongas, el servidor MCP estará disponible en distintos contextos.

Nivel de proyecto
email-mcp/.mcp.json
Solo disponible cuando abres Claude Code desde la carpeta email-mcp/ o desde home-Termux/ (directorio padre).
✓ Ya existe y está configurado
Nivel de workspace raíz
home-Termux/.mcp.json
Disponible desde cualquier subcarpeta del workspace: FinancalFreeHome/, naves-industriales-ai/, etc.
⚠ Necesita la contraseña (ver paso 2)
💡

Recomendación: Configura ambos. Así puedes enviar emails desde cualquier proyecto del workspace, no solo desde la carpeta de email-mcp.

02 Configuración paso a paso

Registrar el servidor MCP

1
Verifica el contenido de email-mcp/.mcp.json
Debe verse exactamente así, con tu contraseña real en SMTP_PASS:
{ "mcpServers": { "email-mcp": { "command": "node", "args": [ "/home/llopez/tablet/Documents/home-Termux/email-mcp/src/index.js" ], "env": { "SMTP_USER": "ldominguez@atisa.com", "SMTP_PASS": "tu_contraseña_aquí", "FROM_EMAIL": "ldominguez@atisa.com" } } } }
⚠️

La ruta en args debe ser absoluta (empezar con /home/...). Rutas relativas no funcionan en MCP.

2
Actualiza también el .mcp.json del workspace raíz
Abre /home/llopez/tablet/Documents/home-Termux/.mcp.json y rellena el SMTP_PASS:
# Ruta: home-Termux/.mcp.json { "mcpServers": { "email-mcp": { "command": "node", "args": ["/home/llopez/tablet/Documents/home-Termux/email-mcp/src/index.js"], "env": { "FROM_EMAIL": "ldominguez@atisa.com", "SMTP_USER": "ldominguez@atisa.com", "SMTP_PASS": "tu_contraseña_aquí" ← esto } } } }
3
Abre Claude Code en el workspace
Desde la terminal, ve a la carpeta del workspace y abre Claude Code:
cd /home/llopez/tablet/Documents/home-Termux claude
Claude Code leerá automáticamente el .mcp.json al arrancar y lanzará el servidor email-mcp en segundo plano.
4
Verifica que el MCP está activo
Dentro de Claude Code, escribe el comando:
/mcp
Debes ver email-mcp en la lista con estado connected. Si aparece en rojo o no aparece, revisa la sección de errores más abajo.
Salida esperada de /mcp
● email-mcp connected
7 tools: email_enviar, email_programar, email_borrador, email_plantilla, email_cola, email_cancelar, email_enviados
5
Haz una prueba de envío
Escribe en Claude Code (en lenguaje natural):
👤
Tú
Manda un email de prueba a ldominguez@atisa.com con asunto "Test email-mcp" y el mensaje "Funciona correctamente"
🤖
Claude
Voy a enviar el email de prueba.
⚙ email_enviar
✅ Email enviado correctamente a: ldominguez@atisa.com
Asunto: Test email-mcp
Revisa tu bandeja de entrada. Si el email llega, todo está funcionando.

FAQ Problemas frecuentes en Claude Code

Si algo no funciona

❌ El comando /mcp no muestra email-mcp
Causas posibles:
1. Abriste Claude Code desde una carpeta que no tiene el .mcp.json en su árbol. Solución: abre desde home-Termux/ o desde email-mcp/.
2. El .mcp.json tiene un error de sintaxis JSON (una coma de más, una llave sin cerrar). Verifica el archivo con: node -e "JSON.parse(require('fs').readFileSync('.mcp.json','utf8'))". Si muestra error, corrige el JSON.
3. La ruta en args es incorrecta. Verifica que el archivo existe: ls /home/llopez/tablet/Documents/home-Termux/email-mcp/src/index.js
❌ email-mcp aparece en rojo o como "failed"
El servidor arrancó pero falló. Para ver el error exacto, corre el servidor manualmente:
node /home/llopez/tablet/Documents/home-Termux/email-mcp/src/index.js
El error aparecerá en stderr. Los más comunes: SMTP_PASS no rellena, ruta de index.js incorrecta, o error de permisos de archivo.
❌ Claude "no encuentra" la herramienta email_enviar
Verifica que en /mcp aparece "7 tools". Si aparece "0 tools" el servidor arrancó pero no expuso las herramientas — indica que src/index.js tiene un error de sintaxis. Corre: node --check email-mcp/src/index.js para verificar.
❌ Claude intenta usar la herramienta pero devuelve error de autenticación
Esto ocurre cuando SMTP_PASS en el .mcp.json está vacío o incorrecto. El servidor arranca sin validar las credenciales (fallo lazy), pero falla al intentar enviar. Verifica que SMTP_PASS tiene el valor correcto en el .mcp.json (no solo en el .env) — Claude Code usa las env del .mcp.json para arrancar el proceso.
❌ El email programado no se envió a la hora indicada
El scheduler verifica la cola cada 30 segundos, pero solo mientras Claude Code está abierto y el servidor MCP está corriendo. Si cerraste Claude Code, el servidor se detiene y los jobs pendientes quedan guardados en queue.json — se enviarán la próxima vez que Claude Code esté activo y llegue la hora programada.

Todo configurado — qué recordar

Para usar email-mcp
  • 📂 Abre Claude Code desde home-Termux/
  • 🔌 El MCP arranca solo — no necesitas hacer nada
  • 💬 Habla en español natural: "manda el reporte a..."

No olvidar
  • 🔒 Nunca subas .env a git — está en .gitignore
  • 🔒 No compartas el .mcp.json con la contraseña
  • ⏱ Los emails programados requieren que Claude Code esté abierto
  • 📧 Verifica spam en los primeros envíos
  • 🔄 Si cambias la contraseña de Outlook, actualiza .env y .mcp.json
